# 共享单车数据挖掘方法说明

> 数据来源：hour_wash.csv（预处理后）
> 挖掘目标：从骑行数据中发现规律，预测需求，理解用户行为

---

## 一、任务概览

| 任务 | 类型 | 方法 | 目标 |
|------|------|------|------|
| 回归预测 | 有监督学习 | 线性回归、岭回归、随机森林、梯度提升、XGBoost | 预测每小时租车量 |
| 特征重要性分析 | 模型解释 | 基于树模型的分裂增益 | 找出影响租车量的关键因素 |
| 聚类分析 | 无监督学习 | K-Means | 自动发现骑行行为的自然分群 |
| 分类任务 | 有监督学习 | 决策树 | 预测需求档位（低/中/高） |

## 二、回归预测

### 任务定义

以 `cnt_log`（租车量对数值）为目标，用 23 个特征训练回归模型，预测任意时段的租车需求。

### 模型选择

#### 线性回归

建立特征与目标之间的线性方程：

```
cnt_log = w₁×hr + w₂×atemp + w₃×workingday + … + b
```

每个系数 w 直接反映该特征对租车量的影响方向和强度，是可解释性最强的模型。适合作为基准线（Baseline），后续所有模型都与它对比。

**适用前提：** 需要对 `season`、`weathersit`、`weekday` 做 One-Hot 编码，消除整数序数假设。

#### 岭回归（Ridge）

在线性回归的损失函数中加入 L2 正则项，对过大的系数施加惩罚：

```
损失 = 残差平方和 + λ × Σwᵢ²
```

当特征之间存在共线性，或特征数量较多时，岭回归比普通线性回归更稳定，不容易过拟合。λ 越大，模型越保守。

#### 随机森林

构建多棵决策树，每棵树在随机抽取的样本和特征子集上训练，最终取所有树预测值的平均作为输出。

- 单棵树容易过拟合，多棵树投票后误差互相抵消
- 对异常值和噪声不敏感
- 天然输出特征重要性

#### 梯度提升（Gradient Boosting）

串行地训练多棵决策树，每棵新树专门拟合上一棵树的**残差**（预测错误的部分），逐步纠正误差：

```
第1棵树 → 预测大致趋势
第2棵树 → 拟合第1棵树的误差
第3棵树 → 拟合前两棵树的累计误差
……
```

比随机森林精度更高，但训练速度较慢，对超参数更敏感。

#### XGBoost

梯度提升的工程优化版本，在损失函数中额外加入正则项控制树的复杂度，并采用并行计算加速训练。在结构化数据上综合表现最强。

### 评估指标

| 指标 | 含义 | 越小越好 |
|------|------|----------|
| MAE（平均绝对误差） | 预测值与真实值的平均偏差 | ✓ |
| RMSE（均方根误差） | 对大误差更敏感，惩罚离群预测 | ✓ |
| R²（决定系数） | 模型解释了多少数据方差，1 为完美 | 越大越好 |

> 所有指标均在 `cnt_log` 空间计算，展示时用 `expm1` 还原为真实租车量。

## 三、特征重要性分析

### 任务定义

利用已训练的树模型，量化每个特征对预测结果的贡献程度，回答：

**"到底是什么因素决定了租车量？"**

### 方法：基于分裂增益的重要性

决策树每次分裂节点时，选择能最大程度降低预测误差的特征。将某个特征在所有树、所有节点中带来的误差下降量累加，即为该特征的重要性得分。

**预期结论：**

| 排名 | 特征 | 原因 |
|------|------|------|
| 高 | `hr`、`hr_sin/cos` | 时间是骑行需求最强的驱动因素 |
| 高 | `workingday` | 工作日与休息日需求模式截然不同 |
| 高 | `atemp` | 体感温度直接影响骑行意愿 |
| 中 | `season`、`mnth` | 季节性周期规律 |
| 低 | `holiday`、`windspeed` | 影响存在但相对有限 |

### 可视化

以横向条形图从高到低排列所有特征的重要性得分，直观展示哪些因素"说了算"。

## 四、聚类分析

### 任务定义

**不使用 `cnt` 标签**，让算法自动在数据中发现骑行行为的自然分组，回答：

**"数据本身存在哪些典型的骑行模式？"**

### 方法：K-Means

将数据点分配到 K 个簇，使每个点到其所在簇中心的距离之和最小：

1. 随机初始化 K 个簇中心
2. 将每个样本分配到最近的簇中心
3. 重新计算每个簇的中心点
4. 重复步骤 2–3 直到簇中心不再变化

**输入特征：** `hr`、`workingday`、`season`、`weathersit`、`atemp`

**K 值选择：** 用肘部法则（Elbow Method）绘制不同 K 值下的总误差曲线，选取曲线斜率开始平缓的拐点。

**预期分群：**

| 簇 | 典型特征 | 含义 |
|------|----------|------|
| A | 工作日 + 早晚高峰 | 通勤型骑行 |
| B | 周末 + 日间 + 温暖 | 休闲型骑行 |
| C | 深夜 / 恶劣天气 | 低活跃时段 |

### 可视化

用 **PCA**（主成分分析）将多维特征压缩为 2 维，绘制散点图，每个点按所属簇着色，直观展示分群结果。

## 五、分类任务

### 任务定义

将连续的 `cnt` 划分为三个需求档位，转化为分类问题：

| 档位 | 范围 | 典型场景 |
|------|------|----------|
| 低需求（0） | cnt < 100 | 深夜、雨天、冬季寒冷时段 |
| 中需求（1） | 100 ≤ cnt < 300 | 日常工作日、季节性骑行 |
| 高需求（2） | cnt ≥ 300 | 工作日早晚高峰、夏季日间 |

### 方法：决策树

从根节点出发，每次选择一个特征和阈值将数据一分为二，递归构建直到叶节点对应一个类别。

```
               是否工作日？
              /            \
           是               否
      小时 < 7？         温度 > 0.5？
      /       \           /       \
   低需求   高需求     中需求    低需求
```

**选择决策树的原因：**
- 分裂规则可以直接翻译为自然语言（"如果是工作日且在早高峰，则高需求"）
- 结果可以画成树状图，适合向中学生直观展示
- 无需特征缩放，对原始特征友好

### 评估指标

| 指标 | 含义 |
|------|------|
| 准确率（Accuracy） | 预测正确的样本占比 |
| 混淆矩阵（Confusion Matrix） | 各类别预测对/错的详细分布 |
| F1 Score | 精确率与召回率的调和平均，适合类别不均衡时使用 |

## 六、方法选择建议

```
目标是预测数值   →  回归预测（优先 XGBoost）
目标是理解规律   →  特征重要性分析
目标是发现模式   →  聚类分析（K-Means）
目标是预测类别   →  分类任务（决策树）
```

教学顺序建议：**决策树分类 → 线性回归 → 随机森林 → 聚类分析 → XGBoost**，从直观到复杂逐步递进。
